# 위험 관리 문서 (Risk Management)

**프로젝트명:** sLLM 기반 HTML DOM 트리 파싱을 활용한 웹 오토메이션 에이전트   
**버전:** v1.0

---

## 1. 위험 관리 개요

본 문서는 프로젝트 개발 및 운영 과정에서 식별된 잠재적 위험 요소를 분류하고, 각 위험의 발생 가능성과 영향도를 평가하여 대응 전략을 정의합니다.

### 위험 등급 기준

| 등급 | 발생 가능성 | 영향도 |
|---|---|---|
| 🔴 **HIGH** | 높음 (70% 이상) | 프로젝트 목표 달성 불가 |
| 🟡 **MEDIUM** | 보통 (30~70%) | 일정 지연 또는 기능 저하 |
| 🟢 **LOW** | 낮음 (30% 미만) | 경미한 품질 저하 |

---

## 2. 기술적 위험 (Technical Risks)

### T-01 | 소형 모델(Gemma-2-2B)의 추론 정확도 한계 🔴 HIGH

| 항목 | 내용 |
|---|---|
| **설명** | 2B 파라미터 모델은 복잡한 페이지 구조에서 올바른 target_id를 선택하지 못하거나 JSON 포맷을 준수하지 않을 수 있다. |
| **발생 가능성** | 높음 — 실제 구현 과정에서 `<a>` 링크를 입력창으로 오인하는 문제가 반복 발생했다. |
| **영향도** | 높음 — 잘못된 액션 선택 시 태스크 실패 또는 무한 루프 진입 |
| **대응 전략** | ① `type` 대상이 `<input>`/`<textarea>`가 아니면 코드 레벨에서 자동 교정 <br> ② JSON 파싱 실패 시 정규표현식 fallback + 최대 3회 재시도 <br> ③ `[TYPEABLE]` 태그를 condensed_text에 명시하여 LLM 혼동 최소화 |
| **잔여 위험** | 매우 복잡한 SPA(단일 페이지 앱) 환경에서는 여전히 오작동 가능 |

---

### T-02 | DOM 구조 변경에 따른 셀렉터 불일치 🔴 HIGH

| 항목 | 내용 |
|---|---|
| **설명** | 대상 웹사이트가 HTML 구조를 변경하거나, 동적 렌더링 타이밍 차이로 인해 `data-agent-id`가 JS와 BeautifulSoup 간에 불일치하는 문제가 발생할 수 있다. |
| **발생 가능성** | 높음 — 네이버 등 대형 포털은 A/B 테스트 및 잦은 프론트엔드 배포로 DOM이 자주 변경된다. |
| **영향도** | 높음 — 셀렉터 불일치 시 Playwright가 올바른 요소를 찾지 못해 타임아웃 발생 |
| **대응 전략** | ① `get_html()` 호출 시 JS로 실제 브라우저 DOM에 먼저 `data-agent-id` 주입 후 HTML 반환 <br> ② `id`, `name` 속성 기반 셀렉터를 우선 사용하고 `data-agent-id`는 fallback으로만 활용 <br> ③ Playwright 타임아웃을 10초로 제한하여 무한 대기 방지 |
| **잔여 위험** | Shadow DOM, iframe 내부 요소는 현재 탐지 불가 |

---

### T-03 | networkidle 타임아웃 🟡 MEDIUM

| 항목 | 내용 |
|---|---|
| **설명** | 광고·트래커 스크립트가 지속적으로 네트워크 요청을 보내는 사이트(네이버 등)에서 `networkidle` 상태에 도달하지 못해 타임아웃이 발생한다. |
| **발생 가능성** | 보통 — 광고가 많은 한국 주요 포털에서 특히 빈번하게 발생 |
| **영향도** | 보통 — 에이전트 동작 자체는 실패하지 않지만 렌더링 미완료 상태에서 DOM을 읽을 수 있다. |
| **대응 전략** | ① `wait_until="load"` + 고정 `asyncio.sleep(1.5)` 방식으로 교체 <br> ② 필요 시 특정 셀렉터 등장 대기 (`wait_for_selector`) 방식 추가 |
| **잔여 위험** | 인터넷 속도가 느린 환경에서는 1.5초 대기가 부족할 수 있음 |

---

### T-04 | 컨텍스트 길이 초과 🟡 MEDIUM

| 항목 | 내용 |
|---|---|
| **설명** | 인터랙션 요소가 매우 많은 페이지(검색결과 468개 감지 사례 발생)에서 condensed_text가 Gemma-2-2B의 컨텍스트 길이(8192 토큰)를 초과할 수 있다. |
| **발생 가능성** | 보통 — 대형 커머스, 뉴스 포털에서 발생 가능 |
| **영향도** | 보통 — 컨텍스트 초과 시 모델이 후반부 요소를 인식하지 못하거나 응답이 잘림 |
| **대응 전략** | ① `MAX_OUTPUT_CHARS = 6,000` 상한으로 텍스트 강제 절단 <br> ② 태스크와 관련 있는 요소를 우선 배치 (검색창, 버튼 최상단) <br> ③ 요소 수가 200개 초과 시 경고 로그 출력 |
| **잔여 위험** | 잘린 컨텍스트로 인해 페이지 하단 요소를 에이전트가 인지하지 못할 수 있음 |

## 3. 환경적 위험 (Environmental Risks)

### E-01 | Ollama 서버 미실행 🟡 MEDIUM

| 항목 | 내용 |
|---|---|
| **설명** | 사용자가 Ollama 서버를 실행하지 않은 상태에서 에이전트를 구동하면 연결 오류가 발생한다. |
| **발생 가능성** | 보통 — 처음 사용하는 사용자에게 빈번하게 발생 |
| **영향도** | 높음 — 서버 미실행 시 에이전트 전체 동작 불가 |
| **대응 전략** | ① `check_server()` 메서드로 에이전트 실행 전 서버 상태 사전 확인 <br> ② 연결 실패 시 "ollama serve 명령을 먼저 실행하세요" 안내 메시지 출력 |
| **잔여 위험** | 자동 서버 재시작 로직은 미구현 |

---

### E-02 | 메모리 부족 (OOM) 🟡 MEDIUM

| 항목 | 내용 |
|---|---|
| **설명** | Gemma-2-2B(INT4 약 2GB) + Playwright Chromium(약 500MB) 동시 실행 시 저사양 기기에서 메모리 부족이 발생할 수 있다. |
| **발생 가능성** | 낮음 — MacBook Pro 14(16GB+) 환경에서는 안정적이나, 8GB 이하 기기에서는 위험 |
| **영향도** | 높음 — OOM 발생 시 OS가 프로세스를 강제 종료 |
| **대응 전략** | ① Playwright `headless=True` 모드로 전환하여 GPU 렌더링 부하 절감 <br> ② Ollama `num_predict=128` 제한으로 KV 캐시 메모리 최소화 |
| **잔여 위험** | 8GB 이하 환경에서는 사용 권장하지 않음 |

---

### E-03 | 대상 웹사이트의 봇 탐지 🟢 LOW

| 항목 | 내용 |
|---|---|
| **설명** | Playwright 자동화 브라우저를 탐지하는 사이트에서 CAPTCHA 또는 접근 차단이 발생할 수 있다. |
| **발생 가능성** | 낮음 — 네이버 등 국내 주요 사이트에서는 현재 탐지되지 않음 |
| **영향도** | 보통 — CAPTCHA 발생 시 에이전트가 이를 해결하지 못하고 멈춤 |
| **대응 전략** | ① `slow_mo=300ms` 설정으로 인간과 유사한 속도로 동작 <br> ② `locale="ko-KR"`, `timezone="Asia/Seoul"` 설정으로 자연스러운 브라우저 환경 구성 |
| **잔여 위험** | 고도화된 봇 탐지 시스템(예: Cloudflare)은 우회 불가 |

---

## 4. 프로젝트 운영 위험 (Operational Risks)

### O-01 | 무한 루프 진입 🟡 MEDIUM

| 항목 | 내용 |
|---|---|
| **설명** | 에이전트가 동일한 액션을 반복 실행하면서 태스크를 완료하지 못하는 상황이 발생할 수 있다. |
| **발생 가능성** | 보통 — 실제 구현 과정에서 10+ 스텝 반복 문제가 관찰됨 |
| **영향도** | 높음 — 무한 루프 시 비용(시간) 낭비 및 사용자 경험 저하 |
| **대응 전략** | ① `MAX_STEPS = 15` 하드 리밋으로 강제 종료 <br> ② 실패한 액션 정보를 LLM 히스토리에 포함하여 반복 방지 <br> ③ URL 변경 감지로 조기 완료 처리 |
| **잔여 위험** | 15 스텝 내에 태스크를 완료하지 못하면 실패로 처리됨 |

---

### O-02 | 에이전트 행동 예측 불가 🟢 LOW

| 항목 | 내용 |
|---|---|
| **설명** | LLM 기반 에이전트는 동일한 입력에도 비결정론적 출력을 생성할 수 있어 행동 예측이 어렵다. |
| **발생 가능성** | 낮음 — `temperature=0.0` 설정으로 결정론적 출력으로 고정 |
| **영향도** | 보통 — 예측 불가 행동은 의도치 않은 클릭 또는 폼 제출로 이어질 수 있음 |
| **대응 전략** | ① `temperature=0.0` 고정으로 재현 가능한 출력 보장 <br> ② 액션 실행 전 스키마 검증 (`_is_valid()`) 필수 통과 <br> ③ 개발 단계에서 `headless=False`로 시각 모니터링 |
| **잔여 위험** | 모델 자체의 확률적 특성상 완전한 결정론은 보장 불가 |

## 6. 향후 개선 방향

1. **모델 업그레이드 경로 확보** — Gemma-2-2B → Gemma-2-9B로 교체 시 T-01, T-02 위험 대폭 감소
2. **Shadow DOM / iframe 지원** — 현재 탐지 불가한 중첩 DOM 구조 처리 로직 추가
3. **태스크 완료 판단 고도화** — URL 변경 외 페이지 텍스트 변화 감지로 완료 조건 확장
4. **CAPTCHA 대응** — 탐지 시 사용자에게 수동 개입 요청하는 인터럽트 메커니즘 추가
